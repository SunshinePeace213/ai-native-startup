### Round 1 — Verdict: changes-requested

- **The promised command names contradict cached official behavior for the exact headless surface used by AC2.** The plan assumes `.claude/commands/c-suite/cpo-intake.md` resolves as `/c-suite:cpo-intake`, but `ai-docs/anthropic/agent-sdk/slash-commands.md` says command subdirectories affect only the description and not the command name; under that contract these files create `/cpo-intake`, `/cpo-prd`, and `/cpo-brief`. Fix: resolve the naming mechanism before implementation, revise spec.md/decisions.md/tasks.md/AC1–AC2 to a documented layout, or define an explicit tested compatibility fallback and run the deterministic command-discovery probe before building the full commands.
- **The persona subagents are not guaranteed to receive the knowledge skills they must follow.** tasks.md has the stage commands load skills in the main session, then delegates drafting to fresh-context subagents, but `ai-docs/anthropic/subagents.md` states that a named subagent does not see skills already invoked by the parent; skill content reaches it only through its own `skills` frontmatter or an explicit invocation/delegation payload. The legacy `disable-model-invocation: true` fallback also cannot be preloaded. Fix: add the relevant `skills` preload/invocation contract to Task 3 and Task 5, and specify a working fallback that passes the required content when preloading is disabled; validate this in AC7/AC8.
- **The PRD review-round contract is internally contradictory.** spec.md promises `≤2 rounds`, decisions.md and Task 5 call the gate a maximum of 2 rounds, while the same gate and AC4 offer “one more round” after round 2. Builders cannot implement both a hard two-round cap and a third round. Fix: consistently define two automatic rounds followed by at most one user-authorized extra round (or remove the extra-round option) across all four files.
- **The locked template scope contradicts itself.** decisions.md rejects `products/_templates/` and places all templates inside their owning skills, but `## Open Questions / Out of Scope` says both `_templates/` and the fixture ship under `products/`. Fix: remove `_templates/` from that out-of-scope line so the only shipped `products/` content is `_example-bluebird-bakery/`.
- **The engagement slug is interpolated into filesystem, branch, worktree, and GitHub operations without a validation requirement.** A malformed or shell-significant `<client-slug>` can escape `products/`, create an invalid/ref-confusing branch, or alter a shell command. Fix: add a canonical safe-slug rule and quoting requirement to decisions.md and the `cpo-intake` preflight task (with an explicit fixture exception if `_example-` remains valid), then add rejection cases to acceptance-criteria.md.
- **The validation commands do not prove several criteria or the objective.** AC1's command omits Purpose, frontmatter, and loaded-skill checks; AC3/AC4 only grep a few words while leaving resume semantics, verdict-file-only handling, and digest upsert untested; AC6 does not check roster classifications; AC11 does not check five named design files, locked requirements, or every stage state. No check covers the objective's engagement issue/branch, stage commits/comments/PR, or lessons updates. Fix: replace the partial greps with deterministic structural/fixture assertions (and recorded GitHub-state checks where required) that cover every clause, preferably in one validation script mapped back to AC1–AC11.

**Recommendations (advisory, non-blocking):**

- Reduce the seven-builder team to 3–5 file-disjoint roles. `ai-docs/anthropic/agent-teams.md` recommends starting in that range and notes that coordination overhead rises with team size; the hub and validation roles do not need dedicated long-lived teammates.
- Add `ai-docs/anthropic/agent-sdk/slash-commands.md`, `ai-docs/anthropic/headless.md`, and `ai-docs/anthropic/agent-teams.md` to `## KB References`; command discovery, `claude -p` validation, and the seven-member build team currently depend on them.
