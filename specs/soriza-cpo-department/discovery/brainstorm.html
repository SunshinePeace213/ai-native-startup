Ten rungs from a one-line worktree fix to a department that runs itself

The harness layer stays the foundation. On top of it: Soriza's first operating department — a CPO/design studio that walks clients down the fidelity ladder (intake → brief → sitemap/IA → wireframe → section briefs) and hands Ringo a brief packet. Every card below is anchored to a file that exists in this repo today. Toggle what resonates; the prompt at the bottom becomes the interview's input.

Ordering is cost, cheapest first — rungs 1–4 are locked-decision groundwork, 5–6 are the gates, 7–10 ladder the two open decisions (org shape, role mapping) from "no new agents" to "an agent team per engagement."

Locked — folded in, not reopened
  • Client home is projects/<client>/; the folder structure is the first deliverable.
  • Agents draft all files (intake → brief packet); Ringo owns assets, templates, and Claude Design prototyping.
  • Department memory = path-scoped .claude/rules/soriza-cpo/ family + a lessons log on the development-log contract.
  • KB mirrors copy into worktrees via .worktreeinclude; official design/UX sources registered via /harness-layer:kb add; doctrine lives in rules, never fake mirrors.
  • Work ships as an epic with child issues — one pipeline run per child.

The open decision the top rungs embody: org shape

Rungs 7, 8, and 10 are the same workflow under three org shapes. Pick by where judgment must live and what a hand-off may lose — context cost and delegation reliability pull in opposite directions.

Flat commands (rung 7)

  • Roles dissolve into phases of chained commands — the discovery chain's proven shape.
  • Zero delegation loss: one session holds intake, brief, and IA context end to end.
  • Limit: session length; five section briefs draft sequentially.
End-to-end: one session walks the rungs, Ringo between gates.

Orchestrator + specialists (rung 8)

  • Roles = deliverables = subagents at depth 1 — nesting not needed (each specialist is a leaf).
  • Five section briefs fan out in parallel; each agent loads only its own doctrine.
  • Cost: the delegation message is the interface — anything not written down is lost at hand-off.
End-to-end: orchestrator command walks the ladder, fans out per deliverable, assembles the packet.

Agent team (rung 10)

  • Persistent teammates + shared task board — task-tools.md already specifies the lead protocol.
  • Survives multi-day engagements and mid-flight client replies.
  • Cost: highest context spend, most moving parts; overkill for a single deliverable.
End-to-end: CPO lead session per engagement; teammates own rungs; board is the audit trail.
