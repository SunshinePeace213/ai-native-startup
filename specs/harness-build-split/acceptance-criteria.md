# Acceptance Criteria: Split harness-build into build + review commands

> The definition of "done" for [spec.md](./spec.md). Every criterion is observable and testable, and
> every task in tasks.md should map to at least one criterion here.

## Acceptance Criteria

- **AC1** — `.claude/commands/harness-layer/harness-build.md` and `.claude/commands/harness-layer/harness-review.md` both exist and their combined `wc -l` is under 132.
- **AC2** — Each command's `## Variables` declares exactly `PATH_TO_PLAN`, `ISSUE_NUMBER`, `PR_NUMBER`, `REVIEW_PROFILE`; neither file contains a Preflight section nor the strings `STAGES:`, `CROSS_CHECK:`, `ROUND_INPUT`, `REQUIRED_PLUGINS`, or `round-N-input`.
- **AC3** — harness-build.md sequences tidy (harness-simplifier / code-simplifier, behavior-preserving) BEFORE opening the draft PR, posts the tidy findings directly as the `<!-- report:tidy -->` PR comment, and contains no held-report rule.
- **AC4** — harness-build.md finishes every repository write before snapshotting the hand-off SHA (`git rev-parse HEAD`), then records `PR #M` + `Hand-off SHA` in spec.md `## Tracking` as one final metadata commit; it STOPs when `## Tracking` has no Issue `#N`; harness-review.md STOPs when `## Tracking` has no PR number. Each file contains exactly one STOP.
- **AC5** — harness-review.md drives Codex via a `sonnet` runner (retry-once, verdict-line grep, digest relay), spawns fixers per model-selection, and caps each invocation at 2 attempts while the report number N stays global (highest existing report + 1); an attempt-2 `changes-requested` posts the final report comment and leaves the PR draft; an `approved` verdict follows the terminal-commit rule — dev-notes render + memory step first, ONE terminal commit, PR head verified equal to it (stage-table Ready evidence), `gh pr ready`, no repository mutation after. No `status:needs-human`, over-cap gate, root-cause rule, Locked Boundaries, or fix-design consult appears in either command.
- **AC6** — `.agents/skills/implementation-review/SKILL.md` contains no Review packet / attestation / `round-N-input` / zero-git machinery; instead Codex runs its own `git diff` over injected SHAs and runs the plan's Validation Commands itself (unexecutable → recorded with reason, blocking; never fabricated); the report keeps the `### Round N — Verdict:` first line and ends with an `**Issue-comment digest:**` paragraph; lens selection, KB grounding, and the confidence filter survive.
- **AC7** — `specs/_templates/implementation-notes.md` is a chronological dev log (a `## Log` of dated entries covering phases, hand-offs, deviations, fixes, lessons) and `.claude/rules/development-log.md` exists flat with no `paths:` frontmatter, a `date · plan · lesson` line format, a ≈40-line cap with distill-on-overflow, and both files state the per-plan vs cross-plan boundary.
- **AC8** — `.claude/rules/harness-layer/artifacts.md` no longer lists the Deviations board row; its inventory has a Dev notes row rendered by harness-review on any verdict; harness-review.md renders `artifacts/dev-notes.html` from implementation-notes.md inside the terminal commit and links it from the PR; no file under `.claude/` or `.agents/` still references the ship brief.
- **AC9** — harness-plan.md's Define Team & Tasks step requires marking tasks whose outcomes must be recorded to memory; both new commands end with a memory step referencing memory-series.md conventions.
- **AC10** — Memory-truth edits landed: AGENTS.md's Core pipeline line includes `/harness-layer:harness-review` and a dev-log pointer bullet exists; git-workflow.md says the draft PR opens after the tidy checkpoint. `REVIEW_PROFILE` gating appears in both commands; harness-ship.md, spec-review/SKILL.md, and `.claude/hooks/` are untouched.

## Validation Commands

Run these to prove the criteria above; each names the criteria it verifies. Set
`B=.claude/commands/harness-layer/harness-build.md` and
`R=.claude/commands/harness-layer/harness-review.md` first. The greps are deterministic
necessary conditions; clauses no grep can prove (prose quality, exact orchestration
semantics) are judged by the Codex implementation-review rounds, which read the files in
full — that residue is stated per command below.

- `[ -f "$B" ] && [ -f "$R" ] && [ $(cat "$B" "$R" | wc -l) -lt 132 ] && echo PASS-AC1` — proves AC1 (both files exist and total < 132; the `-f` guards fail the check when either file is missing).
- `for f in "$B" "$R"; do v=$(sed -n '/^## Variables/,/^## [^V]/p' "$f" | grep -cE '^(PATH_TO_PLAN|ISSUE_NUMBER|PR_NUMBER|REVIEW_PROFILE):'); t=$(sed -n '/^## Variables/,/^## [^V]/p' "$f" | grep -cE '^[A-Z_]+:'); [ "$v" -eq 4 ] && [ "$t" -eq 4 ] && echo "PASS-AC2 $f"; done` — proves AC2's variable clause (exactly the four names, no fifth entry).
- `grep -lE 'Preflight|ROUND_INPUT|round-N-input|REQUIRED_PLUGINS|needs-human|Locked Boundaries|over-cap|[Rr]oot-cause|fix-design consult' "$B" "$R"; [ $? -eq 1 ] && echo PASS-AC2-AC5-forbidden` — proves AC2's and AC5's deleted-machinery clauses (no file matches).
- `w=$(sed -n '/^## Workflow/,/^## [^W]/p' "$B"); t=$(echo "$w" | grep -niE 'simplifier|tidy' | head -1 | cut -d: -f1); p=$(echo "$w" | grep -niE 'draft PR|pr create' | head -1 | cut -d: -f1); [ -n "$t" ] && [ -n "$p" ] && [ "$t" -lt "$p" ] && echo PASS-AC3-order` — proves AC3's ordering clause (tidy step precedes the draft-PR step inside `## Workflow`).
- `grep -q 'report:tidy' "$B" && grep -qi 'behavior-preserving' "$B" && grep -q 'harness-simplifier' "$B" && grep -q 'code-simplifier' "$B" && ! grep -qwiE 'hold|held|holding' "$B" && echo PASS-AC3-content` — proves AC3's tidy-comment, simplifier, and no-held-report clauses.
- `grep -q 'Hand-off SHA' "$B" && grep -q 'rev-parse HEAD' "$B" && [ $(grep -c 'STOP' "$B") -eq 1 ] && [ $(grep -c 'STOP' "$R") -eq 1 ] && echo PASS-AC4` — proves AC4 (hand-off snapshot recorded; exactly one STOP per file — the two Tracking refusals).
- `grep -q 'sonnet' "$R" && grep -qiE 'once' "$R" && grep -q 'digest' "$R" && grep -qE 'terminal commit' "$R" && grep -q 'gh pr ready' "$R" && grep -qiE 'equals|matches' "$R" && grep -qiE '(max|most) (2|two)' "$R" && grep -qi 'attempt' "$R" && grep -qi 'draft' "$R" && echo PASS-AC5` — spot-proves AC5's runner (retry-once, digest relay), 2-round cap, draft hand-off, terminal-commit + head-equality clauses; loop semantics beyond keywords are review-judged residue.
- `[ $(grep -cE 'round-N-input|Review packet|attestation|zero git' .agents/skills/implementation-review/SKILL.md) -eq 0 ] && grep -q 'Issue-comment digest' .agents/skills/implementation-review/SKILL.md && grep -q 'git diff' .agents/skills/implementation-review/SKILL.md && grep -q 'Validation Commands' .agents/skills/implementation-review/SKILL.md && grep -qiE 'cannot (be )?execut|unexecut' .agents/skills/implementation-review/SKILL.md && grep -qE '^### Round N — Verdict' .agents/skills/implementation-review/SKILL.md && echo PASS-AC6` — proves AC6 (packet machinery gone; digest, self-serve diff, self-run validation with unexecutable-blocks rule, verdict format present).
- `grep -q '## Log' specs/_templates/implementation-notes.md && grep -q 'development-log' specs/_templates/implementation-notes.md && test -f .claude/rules/development-log.md && ! head -1 .claude/rules/development-log.md | grep -q -- '---' && grep -q '40' .claude/rules/development-log.md && grep -q 'implementation-notes' .claude/rules/development-log.md && echo PASS-AC7` — proves AC7 (chronological Log section, no-frontmatter rule file, ≈40-line cap, boundary stated in both directions).
- `! grep -q 'Deviations board' .claude/rules/harness-layer/artifacts.md && grep -q 'Dev notes' .claude/rules/harness-layer/artifacts.md && grep -q 'dev-notes.html' "$R" && ! grep -rqiE 'ship.?brief' .claude .agents && echo PASS-AC8` — proves AC8 (row swapped; review renders the page; no ship-brief reference survives under `.claude/` or `.agents/`).
- `grep -qi 'memory' .claude/commands/harness-layer/harness-plan.md && grep -qi 'memory-series' "$B" && grep -qi 'memory-series' "$R" && echo PASS-AC9` — proves AC9 (plan marks memory-worthy tasks; both commands carry the memory step).
- `grep -q 'harness-review' AGENTS.md && grep -q 'development-log' AGENTS.md && grep -q 'tidy checkpoint' .claude/rules/git-workflow.md && grep -q 'REVIEW_PROFILE' "$B" && grep -q 'REVIEW_PROFILE' "$R" && git diff origin/main --quiet -- .claude/commands/harness-layer/harness-ship.md .agents/skills/spec-review .claude/hooks && echo PASS-AC10` — proves AC10 (pointers landed; profile gating present; out-of-scope files untouched).
- `git diff origin/main --name-only -- ':!specs/harness-build-split' ':!ai-docs'` — proves AC10's blast radius. Output lists ONLY the files named in spec.md `## Relevant Files` / `### New Files`.
