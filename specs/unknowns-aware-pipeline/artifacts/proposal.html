The Unknowns-Aware Pipeline

Folding Thariq's field guide — blindspot passes, design directions, implementation notes, and merge quizzes — into plan → build → ship. This page is itself a demo of the medium: change cards ordered by how likely you are to tweak them, chips that assemble your reply, and a three-question quiz at the end.

The idea in one matrix

Every failed agent run traces to a gap between the map (your prompt) and the territory (the codebase, the users, your own taste). Each quadrant has a different cure — and a different home in the pipeline.

Existing steps stay; the unknowns layer threads through the seams. Marked steps are new. Every artifact is a committed file in specs/<name>/artifacts/ and a published interactive page.

C1

Ship brief + merge quiz at end of build

likely to change

On approval the lead authors artifacts/ship-brief.html — leads with what shipped, pre-answers reviewer objections with evidence links, ends in a short quiz (medium/complex plans only). It is committed together with the approval-round report commit, so that one commit is the approved head and the ship-time SHA guard still holds. Delta reviews exclude artifacts/ exactly as they exclude reviews/. The build report says: take the quiz before /harness-ship.

.claude/commands/harness-layer/harness-build.md
C2

Design-directions channel in the Grilling Protocol

likely to change

When a ledger decision is taste-shaped — you'd recognize the right answer but can't specify it (UX, output format, report layout) — the planner renders 2–4 alternatives as an interactive artifact with steal/skip chips and a reply assembler, instead of a text-only question. The choice is still confirmed via AskUserQuestion and recorded in decisions.md, so the decision record stays canonical. The trigger definition of "taste-shaped" is the part most worth tuning.

C3

Blindspot pass before grilling

settled

New workflow step after codebase/KB understanding: report the task's unknown unknowns as cards — what it is, why it matters, proposed resolution — as an artifact for medium/complex tasks, inline text for simple ones. Unresolved cards seed the grilling ledger first, and grilling questions are explicitly ordered by architectural blast radius. Findings land in decisions.md under ## Blindspots.

.claude/commands/harness-layer/harness-plan.md
C4

Implementation notes + builder deviations contract

settled

Build creates specs/<name>/implementation-notes.md at implement start. Every builder's hand-off gains a Deviations field: what diverged from the plan, what forced it, the call made. The lead folds entries in at each checkpoint commit; a deviation touching a locked decision or acceptance criterion stops for an AskUserQuestion gate. The Codex review packet points at the file so divergence is dispositioned, not rediscovered.

.claude/commands/harness-layer/harness-build.md
C5

plan-fidelity lens in harness-review

settled

A seventh reviewer lens: diff versus spec.md, tasks.md, and implementation-notes.md. A divergence from the plan with no implementation-notes entry is a finding — confidence-scored and filtered like every other lens, so it can't pad reports.

.claude/commands/harness-layer/harness-review.md

Deliberately left out

KISS guardrails — each of these was considered and cut.

  • No changes to harness-ship. The quiz is advisory; ship stays a fast, mechanical sonnet/low merge with the same guards.
  • No new hooks. Only the existing spec-completeness hook is updated; no artifact-presence or quiz-state enforcement.
  • No quadrant bookkeeping. Decisions aren't classified into the 2×2 in writing — the matrix picks the technique, then disappears.
  • No separate unknowns.md. Blindspots live in decisions.md; deviations live in implementation-notes.md. Two homes, no third file.
  • No mandatory artifacts on trivial work. Simple chores grill lightly and skip the artifact set entirely, per the existing adaptive-depth rule.
